# 学生端业务流程图

## 1. 文档目的

本文档用于把学生端 MVP 的业务流程图形化，方便产品、研发、测试和业务方对齐：

- 业务怎么走。
- 逻辑怎么判断。
- 哪些角色参与。
- 哪些地方有分支、异常和状态变化。
- 需要哪些系统支撑能力。

本流程图围绕当前教务系统的核心闭环设计：

```text
教师发布作业 -> 学生提交作业 -> AI 初评 -> 教师复核 -> 教师发布结果 -> 学生查看结果
```

## 2. 角色说明

| 角色 | 说明 |
| --- | --- |
| 学生 | 查看作业、提交作业、查看提交状态和最终结果 |
| 教师 | 创建课程班、发布作业、查看提交、复核并发布结果 |
| 系统 | 负责身份校验、权限判断、状态流转、文件处理和数据同步 |
| AI | 对学生提交生成初评建议，仅教师可见 |

## 3. 总业务流程图

```mermaid
flowchart TD
  A([开始]) --> B[教师创建课程班并导入学生]
  B --> C[教师发布作业]
  C --> D[系统向课程班学生开放作业]
  D --> E[学生进入产品]
  E --> F{学生是否已登录}
  F -- 否 --> G[学生登录或身份校验]
  G --> H{是否属于该课程班}
  F -- 是 --> H
  H -- 否 --> H1[提示无权限或联系教师]
  H1 --> Z1([流程结束])
  H -- 是 --> I[展示学生作业列表]
  I --> J[学生进入作业详情]
  J --> K{作业是否可提交}
  K -- 否 --> K1[展示不可提交原因]
  K1 --> K2[仅允许查看详情或结果]
  K2 --> Z1
  K -- 是 --> L[学生填写文本、链接或上传附件]
  L --> M{提交内容是否有效}
  M -- 否 --> M1[提示至少提交一种内容或修正格式]
  M1 --> L
  M -- 是 --> N[学生确认提交]
  N --> O[系统生成提交记录]
  O --> P{是否超过截止时间}
  P -- 是 --> P1[标记为迟交]
  P -- 否 --> P2[标记为已提交]
  P1 --> Q[教师端提交统计更新]
  P2 --> Q
  Q --> R{作业是否启用 AI 初评}
  R -- 是 --> S[AI 生成初评建议]
  R -- 否 --> T[进入人工批阅]
  S --> U{AI 初评是否成功}
  U -- 否 --> T
  U -- 是 --> V[进入待教师复核]
  T --> W[教师复核提交]
  V --> W
  W --> X{教师处理结果}
  X -- 退回修改 --> Y[学生查看退回原因并重新提交]
  Y --> L
  X -- 保存草稿 --> W
  X -- 发布结果 --> AA[系统将结果设为学生可见]
  AA --> AB[学生查看最终分数和教师评语]
  AB --> Z([完成])
```

## 4. 学生提交流程图

```mermaid
flowchart TD
  A([学生进入]) --> B{入口类型}
  B -- 普通入口 --> C[进入学生登录页]
  B -- 作业链接 --> D[记录目标作业]
  D --> C
  C --> E[完成登录或身份校验]
  E --> F{是否有目标作业}
  F -- 有 --> G[进入目标作业详情]
  F -- 无 --> H[进入作业列表]
  H --> I[选择待提交作业]
  I --> G
  G --> J[阅读作业说明和提交要求]
  J --> K{作业状态}
  K -- 草稿或撤回 --> K1[提示作业暂不可访问]
  K -- 已截止且不允许迟交 --> K2[提示已截止不可提交]
  K -- 已发布可提交 --> L[进入提交页]
  K -- 已截止但允许迟交 --> L1[进入提交页并提示迟交]
  L1 --> M[填写文本、链接或上传附件]
  L --> M
  M --> N{是否至少有一种提交内容}
  N -- 否 --> N1[提示补充提交内容]
  N1 --> M
  N -- 是 --> O{附件是否上传完成}
  O -- 否 --> O1[等待上传完成或重试]
  O1 --> M
  O -- 是 --> P[确认提交]
  P --> Q[提交成功页]
  Q --> R[查看提交详情]
  R --> S([提交流程完成])
```

## 5. 教师复核与学生结果可见流程图

```mermaid
flowchart TD
  A([学生提交完成]) --> B[教师端出现提交记录]
  B --> C{AI 初评状态}

  C -- 处理中 --> C1[等待 AI 处理]
  C1 --> C
  C -- 失败 --> D
  C -- 已完成 --> E[教师查看 AI 建议]
  E --> F[教师复核分数和评语]

  F --> G{教师操作}
  G -- 退回修改 --> H[填写退回原因]
  H --> I[学生端显示已退回]
  I --> J[学生重新提交]
  J --> A
  G -- 保存草稿 --> K[教师草稿仅教师可见]
  K --> F
  G -- 发布结果 --> L[校验最终分数和评语]
  L --> M{是否满足发布条件}
  M -- 否 --> M1[提示补充分数或确认评语]
  M1 --> F
  M -- 是 --> N[生成结果发布记录]
  N --> O[学生端结果可见]
  O --> P[学生查看最终成绩和教师评语]
  P --> Q([结果流程完成])
```

## 8. 教师端与学生端联动流程图

```mermaid
sequenceDiagram
  participant T as 教师端
  participant S as 系统
  participant ST as 学生端
  participant AI as AI 初评

  T->>S: 发布作业
  S->>ST: 作业对课程班学生可见
  ST->>S: 学生提交作业
  S->>T: 更新提交统计和提交列表
  S->>AI: 触发 AI 初评
  AI->>S: 返回初评建议或失败状态
  S->>T: 展示 AI 建议或人工处理入口
  T->>S: 教师复核并发布结果
  S->>ST: 学生端结果可见
  ST->>S: 查看最终分数和教师评语
```

## 9. 流程节点说明

| 节点 | 判断逻辑 | 关键状态 | 支撑能力 |
| --- | --- | --- | --- |
| 身份校验 | 是否已登录，是否为当前学生 | 未登录、已登录 | 学生登录、token、回跳 |
| 权限判断 | 学生是否属于作业课程班 | 有权限、无权限 | 学生课程班关系 |
| 作业可见性 | 作业是否已发布且未撤回 | 可见、不可见 | 作业状态控制 |
| 提交判断 | 是否允许提交，是否迟交 | 未提交、已提交、迟交 | 截止时间、迟交规则 |
| 内容校验 | 文本、链接、附件至少一种 | 有效、无效 | 表单校验、上传服务 |
| AI 初评 | 是否启用，是否成功 | 处理中、已完成、失败 | AI 任务、失败兜底 |
| 教师复核 | 教师发布、退回或保存草稿 | 待复核、草稿、退回、已发布 | 批阅工作台 |
| 结果可见 | 教师是否发布结果 | 待批阅、学生可见 | 发布记录、权限过滤 |

## 10. 测试关注点

- 学生未登录访问作业链接，登录后是否能回到目标作业。
- 学生访问不属于自己的作业，是否被拦截。
- 作业草稿、撤回、已截止不可提交时，学生端是否正确提示。
- 学生不填写任何内容提交，是否被阻止。
- 附件上传失败后，表单内容是否保留。
- 截止后提交是否标记为迟交。
- 学生提交后，教师端提交数、未提交数是否同步变化。
- AI 初评失败时，教师是否能进入人工批阅。
- 教师退回后，学生端是否显示退回原因并允许重新提交。
- 教师未发布结果前，学生端是否看不到分数和评语。
- 教师发布结果后，学生端是否只显示最终分数和教师评语，不显示 AI 原始建议。
