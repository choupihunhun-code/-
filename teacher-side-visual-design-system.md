# 教师端视觉设计系统规范

## 1. 设计定位

本规范用于教师端教务/作业管理系统的高保真设计与前端实现。产品面向大学教师，尤其是公共课、专业课、实验课教师，整体视觉应体现：

- 清晰、专业、可信赖
- 信息密度适中，适合长时间办公使用
- AI 是辅助建议，不制造过强自动化感
- 教师最终确认的权威感要高于 AI 建议
- 支持日间模式与夜间模式扩展

设计关键词：克制、稳定、低噪音、任务导向、可扫描。

### 2.2 品牌渐变

指定渐变：`#6366F1 -> #8836E9`

建议只用于品牌识别和少量重点区域，避免大面积铺满后台页面。

```css
--gradient-brand: linear-gradient(135deg, #6366F1 0%, #8836E9 100%);
```

适用场景：

- 登录页品牌区
- 空状态插图背景小面积点缀
- AI 初评能力标识
- 关键数据趋势线或图表主线

不建议用于：

- 大面积页面背景
- 表格、表单、批阅正文区域
- 所有按钮统一渐变
- 卡片背景批量使用

### 2.5 警示色使用规则

- `danger` 只用于错误、删除、AI 失败、不可逆操作。
- `warning` 用于待处理、迟交、低置信度、发布前确认。
- AI 失败不可只用红色标签，应配合文字说明和下一步操作。
- 夜间模式下警示色要提升明度，避免在深色背景中发暗。
- 状态表达不能只依赖颜色，必须有明确文字。

## 3. 圆角、阴影、描边

### 3.1 圆角

| Token | 数值 | 用途 |
| --- | --- | --- |
| `--radius-xs` | `4px` | 表格内小标签、微型控件 |
| `--radius-sm` | `6px` | 按钮、输入框、筛选项 |
| `--radius-md` | `8px` | 卡片、弹窗内分组、列表项 |
| `--radius-lg` | `12px` | 弹窗、抽屉、页面主容器 |
| `--radius-pill` | `999px` | 头像、状态胶囊、计数徽标 |

规则：

- 后台系统默认使用 `8px`，避免大圆角显得娱乐化。
- 按钮与输入框使用 `6px`，保证工具感。
- 只有头像、状态徽标、胶囊筛选允许使用全圆角。

### 3.2 阴影

| Token | 日间模式 | 夜间模式 | 用途 |
| --- | --- | --- | --- |
| `--shadow-xs` | `0 1px 2px rgba(15, 23, 42, 0.06)` | `0 1px 2px rgba(0, 0, 0, 0.3)` | 表单、轻浮层 |
| `--shadow-sm` | `0 4px 12px rgba(15, 23, 42, 0.06)` | `0 6px 16px rgba(0, 0, 0, 0.36)` | 卡片 hover、Toast |
| `--shadow-md` | `0 10px 28px rgba(15, 23, 42, 0.08)` | `0 14px 36px rgba(0, 0, 0, 0.42)` | 页面主卡片、弹窗 |
| `--shadow-lg` | `0 24px 64px rgba(15, 23, 42, 0.22)` | `0 28px 72px rgba(0, 0, 0, 0.56)` | Modal、Drawer |

规则：

- 页面主内容以描边为主，阴影为辅。
- 表格和表单不使用重阴影。
- Hover 阴影只提升一档，不做明显漂浮动画。
- 夜间模式阴影更依赖黑色透明度，同时配合边框区分层级。

### 3.3 描边

- 默认容器描边：`1px solid var(--color-border)`
- 输入框描边：`1px solid var(--color-border-strong)`
- Focus 描边：`1px solid var(--color-primary)`
- Focus 外环：`0 0 0 3px rgba(99, 102, 241, 0.16)`
- Danger Focus 外环：`0 0 0 3px rgba(220, 38, 38, 0.14)`

描边优先级：

1. 普通卡片：浅描边
2. 当前选中任务：主色左边线或主色描边
3. 错误状态：红色描边
4. 禁用状态：浅描边 + 低透明度

## 4. 标准化组件规则

### 4.1 按钮

按钮高度：

| 尺寸 | 高度 | 横向内边距 | 用途 |
| --- | --- | --- | --- |
| Small | `32px` | `10px` | 表格行操作 |
| Medium | `36px` | `12px` | 默认按钮 |
| Large | `40px` | `16px` | 表单主操作、弹窗主按钮 |

按钮类型：

| 类型 | 视觉 | 用途 |
| --- | --- | --- |
| Primary | 主色底白字 | 发布、确认、开始复核 |
| Secondary | 白底描边 | 保存草稿、返回、下载 |
| Text | 无边框文字 | 表格操作、更多、查看 |
| Soft | 主色浅底 | AI 采纳、轻强调操作 |
| Danger | 红色浅底或红色实底 | 删除、退回修改 |

规则：

- 一个操作区最多一个 Primary。
- Danger 操作必须有二次确认，不能只 Toast。
- Loading 状态禁用重复点击。
- 表格内操作优先使用文字按钮，不使用厚重描边按钮。
- 按钮文案必须具体，例如“确认发布成绩”，不写“确定”。

### 4.2 弹窗

尺寸：

| 类型 | 宽度 | 用途 |
| --- | --- | --- |
| Small Modal | `480px` | 简单确认 |
| Medium Modal | `640px` | 表单、发布确认 |
| Large Modal | `860px` | 导入预览、字段确认 |

结构：

1. 标题：说明动作
2. 描述：说明影响范围
3. 内容区：表单、预览或风险说明
4. 底部按钮：取消在左，主操作在右

规则：

- 删除、退回、发布成绩必须使用确认弹窗。
- 发布成绩弹窗必须提示学生只看到教师最终结果。
- 导入弹窗必须包含上传前、解析中、校验结果、确认导入状态。
- 弹窗关闭后不应丢失未保存草稿，除非用户确认放弃。

### 4.3 卡片

卡片类型：

| 类型 | 用途 |
| --- | --- |
| Metric Card | 工作台指标 |
| Task Card | 待办、复核队列 |
| Info Card | 当前范围、课程摘要 |
| AI Card | AI 初评建议 |
| Form Card | 表单分组 |

规则：

- 页面区块不要互相嵌套卡片。
- 卡片标题字号不超过 `16px`。
- 指标卡片数字突出，但不要超过 `32px`。
- 当前选中队列项使用左侧主色条，而不是大面积底色。
- AI 卡片必须与教师最终输入区域视觉分离。

### 4.4 输入框

高度：

- 默认输入框：`38px`
- 多行文本框：最小 `104px`
- 搜索框：`36px`

状态：

- Default
- Hover
- Focus
- Filled
- Error
- Disabled
- Loading

规则：

- Label 必须常驻，不只依赖 placeholder。
- 错误提示靠近字段下方。
- 必填项使用文字或星号说明。
- 长文本字段要有字数限制或示例说明。
- AI 评分标准使用大文本框，并提供示例结构。

### 4.5 状态标签

标签结构：

- 小圆点
- 状态文字
- 可选数字

标签不建议统一使用重背景。默认做法：

- 普通信息：无底色，仅文字 + 小圆点
- 表格状态：浅底色
- 风险状态：浅底色 + 明确文字
- 当前选中：可使用浅底色，但不大面积高亮

## 5. 字体系统

### 5.1 字体家族

```css
font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", "Microsoft YaHei", "PingFang SC", Arial, sans-serif;
```

原则：

- 中文后台界面优先保证系统字体清晰。
- 不使用装饰性字体。
- 数字可继承系统字体，必要时设置 `font-variant-numeric: tabular-nums;`。

### 5.2 字号层级

| Token | 桌面端 | 移动端 | 行高 | 字重 | 用途 |
| --- | --- | --- | --- | --- | --- |
| `--font-display` | `28px` | `24px` | `1.25` | `700` | 登录页主标题 |
| `--font-page-title` | `22px` | `20px` | `1.3` | `700` | 页面标题 |
| `--font-section-title` | `18px` | `17px` | `1.35` | `700` | 区块标题 |
| `--font-card-title` | `16px` | `16px` | `1.4` | `650` | 卡片标题 |
| `--font-body` | `14px` | `14px` | `1.6` | `400` | 正文 |
| `--font-body-sm` | `13px` | `13px` | `1.55` | `400` | 辅助正文 |
| `--font-caption` | `12px` | `12px` | `1.5` | `500` | 标签、说明 |
| `--font-metric` | `28px` | `24px` | `1` | `750` | 指标数字 |

### 5.3 字重规则

- 页面标题：`700`
- 卡片标题：`650-700`
- 正文：`400`
- 按钮：`600`
- 标签：`600`
- 重要数字：`750`

### 5.4 移动端适配

- 页面标题不超过 `20px`。
- 工具栏按钮允许换行或变为上下布局。
- 表格在移动端优先横向滚动，不压缩到不可读。
- 批阅工作台三栏在移动端改为单栏顺序：队列 -> 原文 -> AI/教师复核。
- 文本不使用 viewport width 缩放。

桌面端：

- 页面左右边距：`32px`

移动端：

- 页面左右边距：`14px-16px`
- 模块间距：`12px-16px`
- 顶栏改为自适应高度

| 弹窗内容 | `20px` |
| 抽屉内容 | `20px` |
| Toast | `12px 14px` |

### 6.4 留白规则

- 工作台首屏只放关键指标和待办，不堆满说明。
- 批阅工作台以阅读效率为主，卡片间距不宜过大。
- 表单页面用分组留白建立结构，不使用过多卡片嵌套。
- 状态说明靠近相关操作，避免集中放在页面顶部。

## 7. 专属视觉标识

### 7.1 品牌标识方向

建议建立一个简洁的“AI 初评 + 教师复核”视觉识别，不做复杂 Logo。

核心符号：

- 左侧竖线：代表教师把关、审核边界
- 双点连接：代表 AI 建议与教师确认的关系
- 文档折角：代表作业/提交材料

可组合为：

- 应用图标
- AI 初评模块图标
- 空状态插图元素
- Favicon

### 7.2 AI 专属标识

AI 相关区域统一使用：

- 主识别色：`--color-ai #8836E9`
- 辅助渐变：`#6366F1 -> #8836E9`
- 标签文案：`AI 初评`、`AI 建议`、`需教师复核`

规则：

- AI 建议不得使用“已判定”“最终分”等强裁决文案。
- AI 卡片必须显示“仅教师可见”或“发布前需教师确认”的提示。
- AI 失败状态要有下一步：重新生成、下载原文、转人工批阅。

### 7.3 教师确认标识

教师最终结果相关区域使用更稳定的中性色和成功色：

- 最终分数输入区：中性白底 + 强 Label
- 发布结果按钮：主色 Primary
- 已发布状态：Success
- 发布确认弹窗：明确“学生将看到最终分数和评语”

### 7.4 数据状态视觉语言

| 状态 | 视觉表达 |
| --- | --- |
| 待复核 | Warning 小圆点 + 文字 |
| AI 已完成 | Success 小圆点 + 建议分 |
| AI 失败 | Danger 小圆点 + 失败原因 |
| 已发布 | Success 标签 |
| 未提交 | Danger 或 Warning，视截止时间而定 |
| 迟交 | Warning 标签 + 时间说明 |
| 当前选中 | 主色左边线 + 轻描边 |

## 9. 落地检查清单

本清单基于当前产品决策、教师端业务闭环和 `$awesome-design` 中关于色彩、排版、品牌规范的参考方向整理。后续设计评审、Figma 交付和前端实现都应按此检查。

| 序号 | 检查项 | 当前决策 | 落地规则 |
| --- | --- | --- | --- |
| 1 | 主色使用范围 | 是，主色应克制使用 | `#6366F1` 只用于关键操作、当前选中、主要链接和重要 Focus。不要把所有卡片、标签、背景都做成主色系。 |
| 2 | AI 建议与教师最终结果视觉分区 | 需要 | AI 初评建议使用 AI 标识色或浅紫辅助识别；教师最终分数、教师评语、发布结果区域必须独立成明确的教师确认区。学生侧不得展示 AI 原始建议。 |
| 3 | 每个操作区是否只能有一个 Primary | 看情况，根据设计建议执行 | 默认每个操作区最多一个 Primary；批量操作、复杂表单底部、工作台快捷入口可根据任务优先级调整，但主操作必须明显高于次操作。 |
| 4 | 状态组件设计 | 参考市面实际组件规范 | 状态标签参考 Ant Design、Arco、Semi、Material 等成熟后台组件规范：使用文字 + 色彩 + 小圆点/图标组合，不只依赖颜色；AI 状态和教师复核状态要分开表达。 |
| 5 | 小屏表格 | 可以横向滚动，并设置最小宽度 | 表格容器小屏横向滚动；表格本体设置最小宽度，例如 `min-width: 800px`。小屏至少露出前几列关键信息，让用户知道可以横向滑动。 |
| 6 | 弹窗按钮文案 | 需要具体文案 | 弹窗主按钮必须写具体动作，例如“确认发布成绩”“确认退回修改”“确认导入 167 人”，避免只写“确定”。 |
| 7 | Danger 操作 | 必须二次提示 | 删除、退回修改、发布后撤回、清空数据等 Danger 操作必须二次确认；提示中说明影响范围和不可逆后果。 |
| 8 | 输入框 Label | 需要常驻 Label，但层级不能过高 | 表单字段必须有常驻 Label；Label 使用 `13px-14px`、中等字重、次级文字色，不要比字段内容和页面标题更抢眼。 |
| 9 | 夜间模式对比 | 需要足够对比 | 夜间模式文字、描边、状态色需满足可读性。正文与背景至少达到常规阅读对比，警示信息不能发灰或只靠暗色底表达。 |
| 10 | 工作台待处理事项 | 需要一眼看到 | 工作台第一视觉焦点放在待教师复核、AI 失败、未提交提醒等待处理事项。通过布局、视觉层级、统一字体和颜色，让页面更清楚、更专业、更耐看。 |

### 9.1 工作台视觉层级落地规则

工作台应优先回答教师进入系统后的第一个问题：今天我需要先处理什么？

建议层级：

1. 顶部核心待办：待教师复核、AI 失败、未提交提醒。
2. 次级数据概览：当前课程班、平均提交率、已发布结果。
3. 任务列表：按紧急程度展示待处理作业。
4. 快捷入口：发布作业、管理课程班、进入批阅工作台。

设计规则：

- 不在工作台堆叠完整业务说明。
- 指标数字要醒目，但不使用过大字号造成看板化压迫。
- 待处理事项优先使用 Warning/Danger 状态，而不是统一蓝色。
- 主操作按钮只保留最关键的一到两个。
- 课程、作业、学生三类信息需要用分组标题和留白区分。
- 同一屏内字体层级不超过 4 档，避免标题过多。

### 9.2 表格小屏规则

```css
.table-wrap {
  overflow-x: auto;
}

.table-wrap table {
  min-width: 800px;
}
```

小屏优先展示：

- 学生/课程/作业名称
- 当前状态
- 关键数量或分数
- 主操作

次要字段允许通过横向滚动查看。

### 9.3 表单 Label 规则

```css
.field label {
  color: var(--color-text-secondary);
  font-size: 13px;
  font-weight: 600;
  line-height: 1.4;
}
```

Label 的职责是帮助识别字段，不承担标题作用。页面中不要把 Label 做成大字号、强色块或过重字重。

### 9.4 Danger 二次确认规则

Danger 弹窗必须包含：

- 动作名称
- 影响对象
- 后果说明
- 取消按钮
- 明确的危险操作按钮

示例按钮文案：

- 确认退回修改
- 确认删除课程班
- 确认清空导入结果
- 确认撤回发布结果
