# 教师端页面状态、按钮逻辑、空异常状态与提示文案确认清单

## 1. 文档目的

本文档用于确认教师端 MVP 在进入继续开发前，需要明确的页面状态、按钮逻辑、空状态、异常状态和提示文案。

适用对象：

- 产品确认
- UI 高保真设计
- 前端开发
- 后端接口联调
- 测试用例编写

核心判断标准：

- 页面是否知道自己处于什么状态。
- 按钮点击后是否有明确反馈。
- 空数据时是否给出下一步动作。
- 异常时是否告诉用户原因和解决方式。
- 文案是否避免技术化、避免误导教师。

## 2. 总体确认结论

当前 HTML 原型已经覆盖主链路的基础交互：

- 教师登录入口
- 课程班列表与详情
- 创建课程班弹窗
- 学生导入弹窗
- 作业创建与发布确认
- 作业详情与提交状态
- AI 初评与教师批阅工作台
- 退回修改、保存草稿、发布结果
- 成绩导出、导出中、导出成功、导出失败、无数据导出
- 学生端入口、作业列表、作业提交、提交成功、查看结果
- Toast 反馈与 Modal / Drawer 反馈

但如果按真实产品开发标准，目前还需要补齐：

- 每个页面的空状态。
- 每个表单字段的校验错误状态。
- 每个关键按钮的 disabled / loading 规则。
- 接口失败时的错误文案。
- 权限异常文案。
- 重复点击、重复提交、重复发布的处理规则。
- 页面数据从统一测试数据渲染后的完整联动测试。

## 3. 页面状态确认

### 3.1 全局页面状态

每个页面都需要至少确认以下状态：

| 状态 | 含义 | 页面表现 | 用户动作 |
| --- | --- | --- | --- |
| 默认态 | 有正常业务数据 | 展示列表、指标、表单或详情 | 可进行主操作 |
| 加载态 | 正在请求数据或执行操作 | 骨架屏、按钮 loading、局部加载 | 禁止重复点击 |
| 空状态 | 当前没有业务数据 | 空状态插画或简洁提示 | 引导创建、导入、发布 |
| 异常态 | 数据加载失败或接口失败 | 错误提示、重试入口 | 可重试或返回 |
| 无权限态 | 当前角色不能访问 | 权限提示 | 返回首页或重新登录 |
| 成功态 | 操作已完成 | Toast / 成功页 / 状态标签变化 | 继续下一步 |

### 3.2 教师工作台

默认态：

- 展示当前课程班数、待复核数、未提交数、提交率。
- 今日待办展示最紧急的作业和提醒任务。
- 快捷入口包括发布作业、查看提交、进入批阅。

空状态：

- 无课程班：提示“还没有课程班”，主按钮“新建课程班”。
- 无作业：提示“当前暂无作业”，主按钮“发布新作业”。
- 无待办：提示“今天暂无待处理事项”，保留课程进度入口。

异常状态：

- 工作台数据加载失败：提示“工作台数据加载失败，请重试”。
- 部分模块加载失败：只在对应卡片显示失败，不影响整页可用。

按钮逻辑：

- “处理 X 份复核”：进入批阅工作台。
- “提醒”：发送未提交提醒，成功后 Toast。
- “查看全部”：进入作业详情或提交列表。
- “发布新作业”：进入作业创建页。

## 4. 课程班相关页面

### 4.1 课程班列表

默认态：

- 列表展示课程班名称、学生数、进行中作业、待复核、提交率、操作。
- 操作统一为：详情、学生、更多。

空状态：

- 无课程班时显示“还没有课程班”。
- 主按钮：新建课程班。
- 次按钮：导入示例数据。

异常状态：

- 课程班列表加载失败：显示“课程班加载失败，请重试”。
- 删除失败：显示“课程班删除失败，可能已有作业或学生记录”。

按钮逻辑：

| 按钮 | 点击结果 | 成功反馈 | 失败反馈 |
| --- | --- | --- | --- |
| 新建课程班 | 打开创建课程班弹窗 | 课程班创建成功 | 课程班创建失败，请检查必填项 |
| 批量导入学生 | 打开导入学生弹窗 | 学生导入完成 | 导入失败，请检查文件格式 |
| 详情 | 进入课程班详情 | 无需 Toast | 数据加载失败 |
| 学生 | 打开学生名单抽屉 | 无需 Toast | 学生名单加载失败 |
| 更多 | 打开更多操作弹窗 | 根据操作反馈 | 根据操作反馈 |
| 复制链接 | 复制当前课程班邀请链接 | 邀请链接已复制 | 复制失败，请手动复制 |
| 删除课程班 | 二次确认后删除 | 课程班已删除 | 已有作业或学生，不建议删除 |

### 4.2 课程班详情

默认态：

- 展示学生人数、已发布作业、待复核提交、平均提交率。
- 展示该课程班下的多个作业。
- 展示学生名单摘要。

空状态：

- 无学生：提示“还没有学生名单”，主按钮“导入学生”。
- 无作业：提示“还没有发布作业”，主按钮“发布新作业”。

异常状态：

- 作业列表加载失败：显示局部错误和重试按钮。
- 学生名单加载失败：学生区域显示错误，不影响作业列表。

## 5. 作业发布页面

默认态：

- 教师填写作业标题、说明、截止时间、提交方式、AI 初评规则、附件。
- 支持保存草稿和发布作业。

空状态：

- 无课程班：不能发布作业，提示“请先创建课程班”。
- 无学生：允许保存草稿，但发布前提示“该课程班还没有学生名单”。

字段校验：

| 字段 | 必填 | 错误文案 |
| --- | --- | --- |
| 作业标题 | 是 | 请输入作业标题 |
| 截止时间 | 是 | 请选择截止时间 |
| 课程班 | 是 | 请选择发布课程班 |
| 作业说明 | 是 | 请输入作业说明 |
| 评分标准 | AI 开启时必填 | 开启 AI 初评时必须填写评分标准 |

按钮逻辑：

- 保存草稿：允许字段不完整，但必须有标题或课程班。
- 发布作业：必须通过必填校验。
- 发布确认：弹窗二次确认，说明 AI 只做初评建议。

异常状态：

- 发布失败：提示“作业发布失败，请稍后重试”。
- 草稿保存失败：提示“草稿保存失败，当前内容未保存”。
- 附件上传失败：提示“附件上传失败，请重新上传”。

## 6. 作业详情与提交列表

默认态：

- 展示提交率、待复核、AI 失败、未提交。
- 展示学生提交明细。
- 支持提醒、批量 AI 初评、批量下载、进入复核。

空状态：

- 无提交：提示“暂时还没有学生提交”，主按钮“提醒学生提交”。
- 无未提交：提示“所有学生已提交”，不显示提醒按钮。
- 无 AI 失败：AI 失败筛选为空时显示“暂无 AI 失败记录”。

异常状态：

- 提交列表加载失败：提示“提交列表加载失败，请重试”。
- 提醒失败：提示“提醒发送失败，请稍后重试”。
- 批量下载失败：提示“文件打包失败，请重试或分批下载”。

按钮逻辑：

| 按钮 | 点击结果 | 状态变化 |
| --- | --- | --- |
| 提醒未提交 | 发送提醒 | Toast 成功，不改变提交数量 |
| 批量 AI 初评 | 对选中提交发起 AI 初评 | 按钮 loading，完成后 AI 状态变化 |
| 批量下载 | 打包下载附件 | 成功后提示下载开始 |
| 去复核 | 进入批阅工作台 | 批阅队列按当前作业过滤 |
| 查看已发布结果 | 打开结果详情 | 不改变状态 |

## 7. 批阅工作台

默认态：

- 左侧展示批阅队列。
- 中间展示学生原文。
- 右侧展示 AI 初评建议和教师复核表单。
- AI 建议必须与教师最终分数、评语视觉区分。

队列状态：

- 待复核
- AI 失败
- 已发布
- 全部

空状态：

- 待复核为空：提示“当前没有待复核提交”。
- AI 失败为空：提示“暂无 AI 失败提交”。
- 已发布为空：提示“暂无已发布结果”。

异常状态：

- 原文预览失败：提示“文件暂不支持预览，请下载原文件查看”。
- AI 建议生成失败：提示“AI 初评失败，请人工批阅或重新生成”。
- 发布结果失败：提示“成绩发布失败，请重试”。
- 保存草稿失败：提示“草稿保存失败，请检查网络后重试”。

按钮逻辑：

| 按钮 | 点击结果 | 规则 |
| --- | --- | --- |
| 切换课程作业 | 打开课程作业选择弹窗 | 切换后队列和右侧内容同步变化 |
| 队列筛选 | 切换队列 | 数量和学生卡片同步 |
| 采纳建议 | 将 AI 建议填入教师复核表单 | 不直接发布 |
| 重新生成 | 重新请求 AI 初评 | 按钮 loading，保留教师已填内容 |
| 下载原文件 | 下载学生提交附件 | 预览失败时主入口 |
| 退回修改 | 打开退回确认弹窗 | 必须填写退回原因 |
| 保存草稿 | 保存教师分数和评语 | 不对学生可见 |
| 发布结果并下一份 | 发布当前学生最终结果 | 成功后自动切换下一份待复核 |

## 8. 学生端页面

### 8.1 学生作业列表

默认态：

- 展示待提交、已提交、已发布结果的作业。
- 学生只能看到教师发布后的最终结果。

空状态：

- 无作业：提示“当前暂无需要完成的作业”。
- 无已发布结果：提示“教师尚未发布批阅结果”。

异常状态：

- 学生 token 失效：提示“登录状态已失效，请重新进入课程链接”。
- 作业加载失败：提示“作业列表加载失败，请重试”。

### 8.2 学生提交

按钮逻辑：

- 提交作业：校验文本、附件或链接至少一种提交内容。
- 删除附件：从当前提交草稿中移除附件。
- 提交成功：进入提交成功页，教师端提交数和待复核数应同步变化。

异常状态：

- 超过截止时间：允许提交但标记迟交，或按课程规则禁止提交。
- 重复提交：如果允许覆盖，提示“将覆盖上一次提交”；如果不允许，提示“已提交，不能重复提交”。
- 附件过大：提示“附件超过大小限制，请压缩后重新上传”。

## 9. 成绩导出

默认态：

- 展示导出课程班、作业、可导出结果数量。
- 可选择导出字段。
- 默认不包含 AI 原始建议。

空状态：

- 无已发布或草稿复核结果：提示“暂无可导出成绩”。

异常状态：

- 导出失败：提示“成绩表生成失败，请重试或调整字段”。
- 下载失败：提示“文件下载失败，请重新导出”。

按钮逻辑：

| 按钮 | 点击结果 | 状态变化 |
| --- | --- | --- |
| 确认导出 Excel | 进入导出中 | 显示 loading |
| 后台处理 | 关闭弹窗 | 导出任务继续保留 |
| 下载 Excel | 下载文件 | Toast 下载开始 |
| 查看无数据状态 | 展示空状态 | 用于原型验证 |
| 查看失败状态 | 展示异常状态 | 用于原型验证 |

## 10. Toast 文案规范

### 10.1 成功类

使用场景：

- 登录成功
- 创建课程班成功
- 学生导入完成
- 作业发布成功
- 草稿保存成功
- 成绩发布成功
- 导出开始或下载开始

文案格式：

- 标题：动作结果
- 描述：说明影响范围和下一步

示例：

- 学生导入完成：成功导入 167 人，跳过 1 条重复学号。
- 作业已发布：学生端已可见，系统已生成提交记录。
- 成绩已发布，已进入下一份：当前学生的最终成绩已发布，队列已自动定位到下一位待复核学生。

### 10.2 警告类

使用场景：

- 退回修改
- 迟交
- 未提交提醒
- 可能覆盖已有内容

示例：

- 已退回修改：学生将收到退回原因和重新提交截止时间。
- 提醒已发送：已提醒未提交学生，提醒不会触发 AI 初评。

### 10.3 失败类

使用场景：

- 接口失败
- 文件上传失败
- AI 初评失败
- 导出失败
- 权限不足

示例：

- 导入失败：文件格式不符合模板，请下载模板后重新整理。
- AI 初评失败：系统无法解析该文件，请人工批阅或下载原文件。
- 发布失败：成绩暂未发布，请检查网络后重试。

### 10.4 信息类

使用场景：

- 切换页面
- 切换筛选
- 复制链接
- 当前功能说明

示例：

- 筛选已切换：当前查看待复核，共 38 条。
- 课程邀请链接已复制：学生可通过该链接加入课程班。

## 11. 开发实现规则

### 11.1 按钮防重复点击

所有会修改数据的按钮必须有 loading 和 disabled：

- 创建课程班
- 导入学生
- 发布作业
- 学生提交
- 保存批阅草稿
- 发布结果
- 退回修改
- 导出成绩

### 11.2 状态更新规则

前端状态更新必须来自统一数据源：

- 课程班学生数
- 作业数量
- 提交数
- 未提交数
- 待复核数
- AI 失败数
- 已发布数
- 导出任务状态

禁止同一数字在多个页面手写维护。

### 11.3 权限规则

- 教师 token 只能访问教师接口。
- 学生 token 只能访问学生接口。
- 未登录点击教师功能时进入登录弹窗。
- 学生不能看到 AI 原始建议。
- 教师未发布前，学生不能看到分数和评语。

## 12. MVP 验收清单

进入开发联调前，至少确认以下内容：

- [ ] 每个页面有默认态。
- [ ] 每个列表有空状态。
- [ ] 每个表单有字段错误状态。
- [ ] 每个关键按钮有 loading 状态。
- [ ] 每个接口失败有错误提示。
- [ ] 每个危险操作有二次确认。
- [ ] 学生端看不到 AI 原始建议。
- [ ] 教师发布结果后学生端才显示成绩。
- [ ] 导入、提交、发布、导出的数字能同步变化。
- [ ] Toast 文案能说明结果和下一步。

## 13. 当前原型下一步建议

建议继续补齐以下原型交互：

1. 给课程班、作业、提交列表补完整空状态。
2. 给作业发布表单补字段校验错误状态。
3. 给批阅工作台补 AI 失败队列为空、待复核为空状态。
4. 给学生提交补重复提交、截止后提交、附件失败状态。
5. 把所有页面数字继续接入统一 mockData，避免静态文案和动态数据不一致。
